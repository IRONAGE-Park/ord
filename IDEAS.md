## 1. 목표 핀(Pin) 시스템 — 진행도 트래커 ⭐⭐⭐
**문제**  
지금은 만들고 싶은 유닛이 정해져 있어도 진행률을 한눈에 보기 어렵다. 후보 비교 모달을 매번 열어야 한다.

**아이디어**  
- 사용자가 "이 유닛 만들 거다" 하고 1~3개 핀 (별표 / 압정 토글)
- 화면 상단에 sticky 진행률 바: `카이도 ━━━━━░░░░ 67% (부족 5)`
- deck이 바뀔 때마다 즉시 갱신 → 흔함 추가하면 진행률이 바로 채워지는 시각적 피드백
- CandidatesModal · RecipeTreeModal에 "📌 핀" 버튼 추가
- 100% 도달 시 살짝 애니메이션 + "조합하기" 단축 액션

**구현 요지**  
- `store`에 `pinnedTargetIds: number[]` (영속화)
- `deficitFor(id, 1, deck)` 결과를 sticky 헤더에 표시
- 진행률 = `1 - (totalDeficit / totalLeavesNeeded)` 정도

## 2. 즉시 조합 가능 패널 — 좌측 사이드바 상단 ⭐⭐⭐
**문제**  
지금 패에서 만들 수 있는 결과 유닛을 발견하려면 후보 비교 모달을 열어야 한다.

**아이디어**  
- 좌측 SkillSummary 위에 항상 떠있는 "지금 즉시 조합 가능" 박스
- 한 줄당 유닛 + ⚡ 조합 버튼, 클릭하면 즉시 combine 실행
- 흔함 한 개 더 모았을 때 갑자기 "에이스 가능!" 같은 발견 유도
- 비어 있을 땐 "다음 1개 부족" Top 3 힌트로 대체

**구현 요지**  
- 매 deck 변화마다 `UNITS.filter(u => deficitFor(u.id, 1, deck).total === 0)`
- 단, recipe 있는 유닛만 (즉, level 1과 leaf 제외)

## 3. URL 기반 빌드 공유 ⭐⭐
**문제**  
빌드를 친구·디스코드에 공유할 방법이 없음. 스크린샷밖에 없음.

**아이디어**  
- 현재 deck을 base64나 short-id로 인코딩해 `?d=...` 쿼리에 담음
- "공유 링크 복사" 버튼 → 다른 사람이 열면 동일한 패 상태 자동 로드
- 카페/디스코드에서 빌드 자랑·논의에 매우 유용
- 옵션: 핀 / 필터 상태도 함께 인코딩

**구현 요지**  
- `[id1, count1, id2, count2, ...]` → varint encode → base64
- App 시작 시 location.search 읽고 deck 덮어쓰기
- 길어지면 압축 (LZ-string)

## 4. 조합 비용 시뮬레이터 (자동 조합) ⭐⭐
**문제**  
부족분만 채우면 어떻게 되는지 보려면 직접 일일이 클릭해야 함.

**아이디어**  
- RecipeTreeModal에 "이 경로 따라 자동 조합" 버튼
- 부족한 leaf만큼 가상으로 추가 → 모든 중간 단계 차례로 combine 시뮬
- 결과 메시지: "총 N번의 조합으로 카이도 완성, 흔함 X개 더 필요"
- "실제 적용" 토글: 가상이 아니라 실제 deck에 반영

**구현 요지**  
- `simulateBuild(targetId, count, deck)` → 합법 단계 시퀀스 생성
- 결과 미리보기 + Apply 버튼

## 5. 역추적: "이 흔함으로 뭘 만들 수 있나" ⭐
**문제**  
"우솝 1개 모자란데 우솝 안 쓰는 다른 좋은 후보가 있나?" 하는 의사결정을 도울 도구가 없음.

**아이디어**  
- 덱의 chip이나 유닛 행 우클릭 → "이 재료를 쓰는 모든 결과 유닛"
- 또는 `?` 검색 키처럼 `*` 같은 키로 활성
- 결과: 그 재료가 lowestMaterials에 포함되는 모든 유닛 리스트

**구현 요지**  
- `findUsesOf(materialId)` → recipes에서 lowestMaterials 매칭
- 작은 모달로 카드 나열
